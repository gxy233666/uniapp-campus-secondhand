# 个人实训总结报告

本次《AI 辅助编程与工程化实训》中，我完成了一个校园二手交易平台项目。项目最初从选题开始，我考虑过外卖点餐、失物招领、记账系统、学习打卡等方向，最后选择了校园二手交易平台。这个选题比较贴近大学生日常生活，也适合展示前端页面、后端接口、数据库、用户登录、商品管理和业务流程设计等内容。经过整个开发过程，我对一个应用项目从需求分析、技术选型、功能拆分、代码实现、调试修复到最终部署，有了更加完整的认识。

本项目使用 HBuilderX 和 uni-app 开发前端，后端采用 uniCloud 云对象，数据库使用 uniCloud 云数据库，代码托管在 GitHub 上。项目主要实现了用户注册登录、院校选择、首页商品展示、商品搜索、分类筛选、院校筛选、商品发布、图片上传、商品详情、收藏商品、我的发布、我的收藏、商品编辑、下架、删除、标记已售出以及交易意向等功能。为了让交易流程更加合理，我没有直接加入真实支付功能，而是设计成买家提交购买意向，卖家同意后才展示联系方式，之后双方线下交易。这样既符合校园二手交易的实际场景，也避免了卖家联系方式直接公开造成骚扰的问题。

在开发过程中，我遇到的最大困难是 uniCloud 云数据库相关问题。刚开始创建服务空间后，数据库资源初始化失败，导致创建集合、上传 schema、读取商品列表、发布商品等操作多次出现 InternalError 或 resource exhausted 报错。这个问题一开始让我比较困惑，因为前端页面看起来没有明显错误，但后端接口一直无法正常读写。后来通过分析报错信息、手动创建数据库集合、上传 schema，并联系 uniCloud 官方客服确认问题原因，才逐步解决。这个过程让我认识到，应用开发不只是写页面和代码，还要理解云服务、数据库、接口调用和部署环境之间的关系。

AI 工具在本项目中发挥了很大作用。由于我的编程基础并不算强，很多功能如果完全靠自己从零开始写，会比较困难。通过 AI 辅助，我能够先明确功能拆分和实现顺序，再逐步完成页面结构、云对象接口、数据库 schema、注册登录、商品管理、收藏、交易意向、H5 部署和项目文档等内容。在遇到报错时，AI 也能帮助我分析问题出现的位置，比如区分是前端逻辑问题、云数据库问题、schema 问题，还是部署后的 H5 访问问题。尤其是在后期，AI 帮我进行了 Code Review，指出了项目文档滞后、身份校验不够完善、schema 权限偏开放、云存储图片需要 H5 验证等问题，这些建议让我对工程化开发有了更清晰的理解。

通过这次实训，我也意识到 Git 提交记录的重要性。以前我可能会习惯一次性写很多代码，最后统一提交，但这次按照老师要求，每完成一个功能点就进行一次 commit。这样不仅让开发过程更加清楚，也方便回顾每个阶段完成了什么内容。比如初始化项目结构、完善商品数据闭环、实现登录注册、增加院校筛选、完善交易意向、优化 UI、补充文档和部署 H5 等，都有对应的提交记录。这让我理解到 Git 不只是保存代码的工具，也是记录项目开发过程的重要方式。

总的来说，这次项目让我从单纯“写代码”进一步接触到了“做项目”。我不仅完成了一个可以在线访问的校园二手交易平台，也补充了 README、API 文档、Prompt 日志、Code Review 报告和 H5 部署地址等交付材料。虽然项目还有可以继续完善的地方，比如更正式的身份认证、更严格的权限控制、更稳定的云数据库读写、更精致的 UI 设计等，但对于本次实训来说，我已经完成了一个较完整的前后端应用闭环。通过这次实践，我对 AI 辅助编程的理解也更加真实：AI 不能完全代替开发者思考，但可以帮助我更快地理解问题、拆解任务、生成代码和修复错误。最终项目能完成，离不开我自己持续测试、判断和调整，也离不开 AI 工具在开发过程中的辅助。
